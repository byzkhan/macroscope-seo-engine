# AEO (Answer Engine Optimization) Rules

## Direct Answer Positioning
- Answer the article's core question within the first 2 paragraphs
- Use a clear, declarative statement — not a question or conditional
- Target 40-60 words for the direct answer (featured snippet length)
- Front-load the answer before the explanation

## FAQ Section
- Required for every article
- Minimum 4 questions, ideally 5-8
- Each answer: 40-60 words, direct and complete
- Answers must stand alone without surrounding context
- Use "What", "How", "Why", "Does", "Can" question starters
- Match questions to People Also Ask data when available

## Structured Answer Patterns
- Use "[Topic] is..." patterns for definitional queries
- Use "To [action], you need to..." patterns for procedural queries
- Use numbered lists for step-by-step processes
- Use comparison tables for "X vs Y" queries

## Entity Coverage
- Mention key entities (tools, companies, standards) relevant to the topic
- Use full names on first reference, abbreviations after
- Link entities to authoritative sources where appropriate
- Cover entities that appear in knowledge panels for related queries

## Conversational Query Targeting
- Write naturally — AI models prefer conversational, clear prose
- Answer implicit questions within the content (not just the FAQ)
- Use question-and-answer subheading patterns where natural
- Avoid jargon that AI models may not associate with the query

## Citation Readiness
- Structure claims so AI models can cite them directly
- Use precise, quotable statements rather than vague generalities
- Include data points and specific examples that support claims
- Attribute claims to sources where possible
